üöÄ Getting Started

Installation

  1. Install XHRScribe from the Chrome Web Store
  2. Pin the extension to your toolbar for easy access
  3. Click the extension icon to open the popup
  4. Configure your AI provider settings (see Settings)

First Recording

üí° Quick Start: You can start recording immediately without any configuration. Generated tests will use mock data until you set up an AI provider.
  1. Open any website with API calls
  2. Click the XHRScribe extension icon
  3. Click "Start Recording" üî¥
  4. Interact with the website (login, browse, submit forms)
  5. Click "Stop Recording" ‚èπÔ∏è when done
  6. Your session will be automatically saved

üé• Recording API Requests

What Gets Recorded

  • HTTP Requests: URL, method, headers, body data
  • HTTP Responses: Status codes, headers, response body
  • Timing Information: Request duration, timestamps
  • API Metadata: Content types, authentication details

Recording Best Practices

üéØ Focus on API Workflows

Record complete user journeys that involve API interactions (login ‚Üí browse ‚Üí purchase).

üßπ Clean Sessions

Start recording after the page loads to avoid capturing initial page load requests.

üìù Meaningful Names

Rename your sessions with descriptive names like "User Registration Flow" or "E-commerce Checkout".

Session Management

  1. View Sessions: Click "Sessions" tab to see all recorded sessions
  2. Rename Session: Click the edit icon next to session name
  3. Delete Session: Click the delete icon to remove unwanted sessions
  4. View Requests: Click on any session to see captured API requests
‚ö†Ô∏è Data Privacy: Sessions are stored locally in your browser. Sensitive data is automatically masked, but always review captured data before sharing.

ü§ñ AI Provider Setup

  3. Copy the key and paste it in XHRScribe settings
  4. Choose from GPT-4, GPT-4-turbo, or GPT-4o models

  4. Choose from Claude-3.5-sonnet, Claude-3-haiku, or Claude-3-opus

Google (Gemini Models)

  1. Visit Google AI Studio
  2. Create an API key
  3. Configure the key in XHRScribe settings
  4. Use Gemini-1.5-pro or Gemini-1.5-flash models

üîß Troubleshooting

Common Issues

üö´ Recording Not Working

  • Check Permissions: Ensure XHRScribe has debugger permission
  • Refresh Page: Reload the target website after starting recording
  • HTTPS Only: Recording only works on HTTPS websites
  • Ad Blockers: Some ad blockers may interfere with recording

üîë API Key Issues

  • Invalid Key: Verify your API key is correct and active
  • Quota Exceeded: Check your usage limits on the provider dashboard
  • Billing Issues: Ensure your payment method is valid
  • Model Access: Some models require special access approval

üßæ Generation Problems

  • Empty Sessions: Ensure your session contains API requests
  • Large Sessions: Very large sessions may timeout - try filtering
  • Network Issues: Check your internet connection
  • Provider Errors: AI provider may be experiencing downtime

Reset and Recovery

üîÑ Reset Options: If you encounter persistent issues, try these reset options in Settings ‚Üí Advanced:
  • Reset Corrupted Data: Clears encrypted data that may be causing errors
  • Reset to Defaults: Restores all settings to factory defaults
  • Clear Cache: Removes cached AI responses and temporary data

Debug Mode

Enable debug mode in Advanced Settings to see detailed logging in the browser console:

  1. Open Chrome DevTools (F12)
  2. Go to Console tab
  3. Enable debug mode in XHRScribe settings
  4. Reproduce the issue
  5. Check console for detailed error messages
